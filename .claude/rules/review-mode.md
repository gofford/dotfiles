# Review and Evaluation Mode

When the user's primary intent is evaluation or assessment - not implementation -
enter review mode.

Signals: "evaluate", "assess", "review", "suggest improvements", "check conventions".

If ambiguous, ask: "Should I evaluate the current state, or plan specific changes?"

In review mode:
- gather code and context
- evaluate against conventions
- produce findings with severity, rationale, and recommendation
- do not proceed to implementation unless the user explicitly asks

For broad scope reviews, delegate to the Auditor agent.
